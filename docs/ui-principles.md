# UI Principles

## North star

Cool, calm, training-first.

Platform Buddy v2 should look appropriate when opened between sets in a gym.

## Visual tone

Use a restrained training aesthetic:

- Warm gray or charcoal base
- High contrast text
- Clear card hierarchy
- Minimal accent colors
- Strong numbers
- Short labels
- No decorative overload

The UI may use a Shiba Buddy icon, but the interface should not become cute-first.

## Screen hierarchy

Every screen must have:

1. A single primary question
2. A primary action area
3. Secondary details below or inside accordions
4. One concise Buddy note at most

## Mobile rules

Primary review width: 390px.

- Inputs must be thumb-friendly.
- Avoid fixed panels that cover content.
- Do not place persistent controls over form fields.
- Avoid horizontal scrolling unless it is intentional and clearly indicated.
- Avoid large hero art above practical actions.
- Keep first-screen content useful without scrolling.

## HOME rules

HOME should not be a marketing page.

First visible content should answer:

- What should I do today?
- What is the next action?

## LOG rules

LOG must be the fastest screen.

Required first-area order:

1. Lift
2. Weight
3. Reps
4. RPE
5. Save

Memo is useful but secondary.

Forbidden in LOG:

- Video upload
- Camera
- Motion analysis
- VBT
- Large learning content above the form

## PLAN rules

PLAN shows today's work first.

Program settings and long explanations must not appear before today's prescription.

## DATA rules

DATA is a judgment screen, not a chart gallery.

The first section should summarize what the data means.

Charts may appear below judgment.

## MEET rules

MEET is a readiness tool.

Show D-day, checklist, opener candidates, and white-light reminders before long rule content.

## Shiba Buddy usage

Use Buddy as a quiet spotter.

Good:

- Small avatar next to one sentence
- Short practical warning
- Short encouragement after save

Bad:

- Large mascot panels
- Multiple character states per screen
- Background character images
- UI that depends on character art

## Gamification limit

Allowed:

- Weekly completion
- Streak count
- PR badge
- Meet readiness percentage
- White-light quiz progress

Not allowed:

- XP systems
- Leveling
- Gacha
- Character growth
- Daily mission spam
- Reward screens that interrupt logging
